# Feature Plan: feature-showcase

## 개요
홈 랜딩 페이지의 "KEY FEATURES" 섹션을 현재 3개 카드에서 실제 앱의 핵심 기능 6개를 보여주는 풍부한 Bento Grid로 개선한다.

## 배경 및 목적
- **현재 상태**: 3개 카드 (재고 차감 / 데이터 정규화 / FAIL·발주)
- **문제점**:
  - 수술 통계·임상 분석, 재고 실사, 대시보드 알림 등 실제 기능이 마케팅에 미노출
  - 모든 카드가 아이콘 + 텍스트만 → 시각적 임팩트 부족
  - 3열 균일 레이아웃 → 기능 간 중요도 위계가 없음
- **목표**: 전환율 향상을 위해 앱의 실제 가치를 풍부하게 전달

## 사용자 스토리
- 방문자로서 실제 앱 화면을 보거나 핵심 기능을 명확히 파악하여 가입 결정을 내릴 수 있다.
- 치과 원장·실장으로서 "우리 병원에 딱 필요한 기능이 있다"는 확신을 갖고 CTA를 클릭할 수 있다.

## 기능 요구사항

### FR-01: 카드 6개로 확장
현재 3개 → 6개 카드. 기능별 배치:

| 번호 | 제목 | 핵심 copy | 배지 |
|------|------|-----------|------|
| 1 | 실시간 재고 & 자동 차감 | 수술 기록 업로드 → 재고 즉시 반영. 브랜드·사이즈별 현재고 한눈에 | `가장 인기 있는 기능` |
| 2 | 수술 통계 & 임상 분석 | 월별 트렌드, 제조사 점유율, 식립 위치까지. 12개월 데이터 자동 집계 | `NEW` |
| 3 | FAIL 완전 추적 | 수술 중 FAIL → 교환 접수 → 입고까지 단계별 관리. 브랜드별 FAIL률 자동 계산 | — |
| 4 | 스마트 발주 추천 | 소모 패턴 기반 적정 재고 자동 계산. 원클릭 발주로 실수 제거 | — |
| 5 | 재고 실사 & 불일치 감지 | 실물↔시스템 재고 비교. 오차 즉시 확인, 이력 관리 | — |
| 6 | 스마트 데이터 정규화 | 14개 브랜드 표준 지원. 파편화된 표기 자동 통일, 오타 자동 수정 | — |

### FR-02: Bento Grid 레이아웃
- **카드 1 (히어로)**: 좌측 2행 차지 (2배 크기), 인디고 그라디언트 배경 유지
- **카드 2–5**: 우측 2×2 그리드 배치
- **카드 6**: 하단 풀 와이드 or 2열 마지막 카드
- 모바일: 모든 카드 1열 스택, 카드 1이 먼저 등장

```
Desktop (md 이상):
┌──────────────┬──────────┬──────────┐
│              │  카드 2  │  카드 3  │
│   카드 1     ├──────────┼──────────┤
│  (2배 높이)  │  카드 4  │  카드 5  │
├──────────────┴──────────┴──────────┤
│              카드 6 (와이드)         │
└────────────────────────────────────┘
```

### FR-03: 카드 내부 구성
각 카드 (카드 1 제외):
- 아이콘 (현재 방식 유지)
- 제목 (bold)
- 설명 (2–3줄, 구체적 수치 포함)
- 선택적: 미니 stat 칩 ("14개 브랜드", "12개월", "30초" 등)

카드 1 (히어로):
- 기존 요소 유지 + 배지
- 하단에 미니 stat 2–3개 추가 ("수술 업로드 후 30초", "14개 브랜드 지원" 등)

### FR-04: 섹션 헤더 유지
- `KEY FEATURES` 라벨 + `병원 운영의 품격을 높이는 기능` 제목 유지
- 부제 1줄 추가: "치과 임플란트 관리의 모든 것을 하나로"

## 비기능 요구사항
- **반응형**: 모바일 우선. sm: 1열, md: 기존 3열 유지 가능, lg: 새 bento 레이아웃
- **애니메이션**: 카드 hover 시 shadow + translate-y (현재 패턴 유지)
- **접근성**: 카드 role, aria-label 유지
- **성능**: 신규 이미지/스크린샷 없음 (아이콘 기반 유지) → 번들 증가 없음

## 범위 제외 (Out of Scope)
- 탭형 피처 쇼케이스 (스크린샷 포함) — Phase 2로 분리
- 실제 앱 스크린샷 삽입 — 별도 작업
- 다른 랜딩 섹션 변경 (Hero, Stats 등)

## 수용 기준
1. `npm run build` 통과
2. 데스크톱에서 6개 카드가 Bento Grid로 렌더링
3. 모바일(390px)에서 1열 스택으로 렌더링, 카드 1이 최상단
4. 카드 hover 인터랙션 동작
5. 기존 CTA (무료 분석, 회원가입 등) 동선 유지
