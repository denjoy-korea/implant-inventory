## 배경 및 목적

### 문제 정의

1. **플랜 간 차별화 부족**: Basic이 너무 많은 기능을 제공해 Plus 전환 유인이 약함
2. **Free 포지셔닝 불명확**: 재고관리 대시보드까지 Free에 포함되어 제품 진입 전략이 모호
3. **기능 게이팅 구현율 72.2%**: UI 표시 기능 5개가 실제 코드에서 게이팅되지 않음
4. **데이터 불일치**: pricingData.tsx의 기능 목록이 PLAN_LIMITS와 독립적으로 관리되어 변경 시 불일치 발생
5. **결제 UX 문제**: 계좌이체 미가맹 상태에서 선택 시 토스 실패 화면 노출
6. **미구현 기능 표기**: 원클릭 발주, 자동 재고 알림, 연간 리포트 등이 미구현이나 요금제 페이지에 표시

### 목표

- 플랜별 역할과 차별점을 명확히 하여 전환율(CVR) 최적화
- 코드 게이팅과 UI 표기를 100% 일치시켜 사용자 신뢰 확보
- 신규 기능(간편발주, 원클릭 발주) 구현 포함
- 가격 재조정으로 시장 경쟁력 강화

## 신규 플랜 스펙 정의

### 플랜 포지셔닝

| 플랜 | 타깃 | 포지셔닝 |
|------|------|----------|
| **Free** | 소규모 치과 | 수술기록 통계 뷰어 — 재고·발주 없음, 3개월 통계로 맛보기 |

## 구현 범위

### Phase 1 — 코드 기반 수정 (즉시)

#### 1-1. types/plan.ts 변경
- `PLAN_LIMITS` 수치 변경: maxItems (50/150/300/∞), maxUsers (1/1/5/10), retentionMonths (24/24/24/24)
- `PLAN_PRICING` 변경: Basic 27,000/21,000, Plus 59,000/47,000
- `PlanFeature` enum 추가: `order_optimization`, `simple_order`, `exchange_analysis`, `surgery_chart_advanced`, `surgery_chart_basic`
- 새로운 제한 필드 추가: `uploadFrequency: 'monthly' | 'weekly' | 'unlimited'`

#### 1-2. planService.ts 변경
- `canUploadSurgery`: Basic = 주 1회 (캘린더 기준 7일) 로직 추가
- `canViewData(plan, date)`: 조회 가능 날짜 범위 계산 함수 추가
- useFileUpload.ts dead code 제거 ("베이직 플랜 주 1회" 에러 메시지)

#### 1-3. pricingData.tsx 전면 개편
- `plans` 배열: PLAN_PRICING import, 차별화 기능만 나열 (누적 제거)
- `comparisonCategories`: 신규 기능 매트릭스 반영
- 가격 참조: `PLAN_PRICING`에서 동적으로 가져오기 (하드코딩 제거)
- `formatPrice` 중복 제거 (pricingData.tsx 버전으로 통일)

#### 1-4. FeatureGate 추가 (게이팅 누락 수정)
- `return_management`: ReturnManager 진입 시 Plus+ 게이팅
- `order_optimization`: OptimizeModal Plus+ 게이팅
- `exchange_analysis`: 교환관리 하단 4섹션 Plus+ 게이팅
- `surgery_chart_advanced`: 수술기록 나머지 차트 Plus+ 게이팅
- `surgery_chart_basic`: 월별 추세 + 요일별 패턴 Basic+ 게이팅

#### 1-5. 화면 접근 제어
- **Free**: 재고관리 마스터, 주문관리 마스터, 교환 관리, 재고 실사 메뉴 → 잠금 표시 또는 업그레이드 유도 UI
- **Basic**: 규격별 사용량 분석, 교환 관리 하단 4섹션, 수술기록 고급 차트 → 인라인 잠금 처리
- 수술기록 조회 범위 필터링: `canViewData()` 기반으로 조회 날짜 clamp

#### 1-6. 결제 UX 수정
- `PricingPaymentModal.tsx`: 계좌이체 버튼 비활성화 + "준비 중" 툴팁 (토스 정식 가맹 전)

### Phase 2 — 신규 기능 구현

#### 2-1. 간편발주 (Plus+) — 카톡 복사 발주
- 주문관리에서 제조사별 부족 품목 필터링
- [복사] 버튼 → 클립보드 텍스트 생성 (제조사명 + 품목 + 수량)
- 붙여넣기 후 카톡으로 바로 발주 가능한 포맷

#### 2-2. 원클릭 발주 (Business) — 핵심 차별 기능
- 거래처 연락처 DB 연동
- 부족 품목 자동 감지 → 발주서 자동 생성
- 발주 전송 (이메일/문자) + 수령 확인 자동화

#### 2-3. Business 추가 사용자 과금
- 10명 초과 사용자 등록 시 별도 결제 플로우
- 월/연 구독에 합산 또는 별도 인보이스 방식 결정 필요

---

### Phase 3 — 요금제 페이지 UI 개편

- 하위 플랜 대비 **차별화 기능만** 카드에 표시 (누적 기능 목록 제거)
- "출시 예정" 배지: 간편발주, 원클릭 발주 (미구현 기간 동안)
- 계좌이체 준비 중 표시
- 한정 수용 병원 수 (plans[].limit) — DB RPC 동기화 또는 관리 화면 신설
