# Plan: 밸류에이션 범위 축소 실행계획 (Workstream 3/4/5)

> 기준일: 2026-03-04 (KST)
> 목적: 공정가치 범위를 좁히기 위해 불확실성(데이터 표본, 릴리즈 리스크, 증빙 신뢰성)을 4주 내에 구조적으로 축소한다.

---

## 1. 목표와 성공 기준

### 1.1 최종 목표
- 현재 넓은 밸류에이션 범위의 원인(트래픽 표본 부족, 게이트 실패, 증빙 분산)을 해소해 투자/파트너 커뮤니케이션에 사용 가능한 수치 기반 패키지를 확보한다.

### 1.2 정량 성공 기준 (Definition of Success)
1. Workstream 3 (퍼널 집계)
- 최근 30일 유효 세션: 300+
- session_id 누락률: 1% 미만
- 일별 KPI 스냅샷: 28일 중 27일 이상 누적 생성

2. Workstream 4 (릴리즈 리스크)
- `npm run verify:premerge` 3회 연속 Green
- `npm run verify:release` 1회 Green
- 트라이얼 정책(기본 14일 / 베타 신청분 28일) 단일 소스 정렬 완료

3. Workstream 5 (상업/법무 증빙)
- 유료 고객 단위 계약-청구-수금 링크 100% 추적 가능
- 환불/해지 건 근거 문서 100% 연결
- 보안/컴플라이언스 증빙 최신본(작성일 포함) 패키지화 완료

---

## 2. 범위

### In Scope
- Workstream 3: 퍼널 이벤트 집계 자동화와 데이터 품질 관리
- Workstream 4: 타입체크/테스트/빌드 게이트 복구 + 정책 불일치 해소
- Workstream 5: 데이터룸 구조화(상업/법무/보안 증빙)

### Out of Scope (이번 4주)
- 신규 대규모 기능 개발(예: 결제 PG 신규 연동 대공사)
- 가격 정책 자체 변경(Starter 신설 등)
- 장기 코호트(90일 이상) 확정치 도출

## 3. Workstream 3: 퍼널 표본·집계 자동화

### 3.1 현재 기준선
- 최신 스냅샷 기준(2026-02-22 생성): 고유 세션 13, Auth Complete 0

### 3.2 실행 항목
1. 이벤트 사전 동결
- 퍼널 스테이지(landing/pricing/auth/analyze/contact/waitlist/payment) 정의 고정
- 이벤트 명칭/필수 필드(session_id, created_at, page) 검증 규칙 문서화

2. 집계 자동화
- 일별 스냅샷 생성 루틴 표준화 (`scripts/admin-traffic-snapshot.mjs --daily`)
- 리포트 저장 위치 통일: `docs/04-report/traffic-kpi-daily/`

3. 데이터 품질 점검
- 누락률, 중복 이벤트율, 세션 분해 실패율 주간 리뷰
- 이상치 탐지 규칙(급격한 0값/스파이크) 체크리스트 운영

4. 유입 분모 확대 실험 (운영)
- 최소 2개 채널 동시 운영(예: 파트너 유입 + 광고)
- 채널 태깅 규칙 통일로 전환 귀속 가능 상태 확보

### 3.3 산출물
- 일별 KPI 스냅샷 파일
- 주간 요약 리포트 (세션/퍼널/CVR/이상치)
- 이벤트 계측 품질 점검표

## 4. Workstream 4: 릴리즈 리스크 제거

### 4.1 현재 기준선
- `npm run typecheck`: 실패 1건
- `npm run test`: 100개 중 1개 실패

### 4.2 실행 항목
1. 즉시 실패 복구
- TypeScript 컴파일 블로커 해소
- 모바일 핵심 동선 테스트 계약(코드 vs 테스트) 동기화

2. 트라이얼 정책 단일화
- 원칙: 기본 14일, 베타 신청분(컷오프 전 시작) 28일
- 프론트/관리자/서비스/SQL의 계산 기준 일치

3. 품질게이트 반복 검증
- premerge 3회 연속 실행 및 결과 보관
- release 게이트 1회 실행 및 결과 보관

4. 운영 스모크 기준선 확정
- 보안/결제/권한 핵심 경로 체크리스트 재점검

### 4.3 산출물
- 게이트 실행 로그 묶음
- 트라이얼 정책 정렬 변경내역
- 릴리즈 준비 판정서 (Go/No-Go)
