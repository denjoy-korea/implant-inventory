## 1. 배경 및 목적

### 현재 상황
- 임플란트 재고관리 웹서비스가 프론트엔드 전용(localStorage)으로 동작 중
- 병원별 데이터 격리, 역할 기반 권한(Master/Staff), 8개 핵심 기능 모듈 구현 완료
- 서버/DB 미연동 상태로 SaaS 전환 시 가격정책 수립 필요

### 목적
- 서비스 상용화를 위한 요금제 체계 설계
- 병원 규모별 차등 가격 모델 수립
- 무료 체험 → 유료 전환 퍼널 설계

## 2. 타겟 고객 분석

### 주요 고객 세그먼트

| 세그먼트 | 규모 | 특징 | 추정 비율 |
|---------|------|------|----------|
| **소형 치과** | 유닛 1~3대 | 원장 1인 운영, 월 수술 10~30건 | 60% |
| **중형 치과** | 유닛 4~7대 | 원장 + 스태프 2~5명, 월 수술 30~80건 | 30% |
| **대형/네트워크** | 유닛 8대 이상 | 다지점, 스태프 5명 이상, 월 수술 80건+ | 10% |

### 핵심 가치 제안
- 수술기록 기반 자동 재고 추적 (수동 재고실사 제거)
- 14개 제조사 사이즈 포맷 자동 정규화 매칭
- FAIL 교환 추적 및 발주 최적화
- 치과 경영 데이터 시각화

## 3. 요금제 체계

### 3.1 플랜 구성

| 항목 | 내용 |
|------|------|

| **목적** | 서비스 체험, 소형 치과 유입 |
| **계정 수** | Master 1명 (Staff 불가) |

## 4. 가격 산정 근거

### 4.1 경쟁사 분석

| 서비스 | 유형 | 가격대 | 비고 |
|--------|------|--------|------|
| 덴트웹 | 치과 통합관리 | 월 50,000~150,000원 | 예약+차트+재고 포함 |
| 덴탈마스터 | 치과 경영관리 | 월 30,000~100,000원 | 예약 중심 |
| 일반 SaaS 재고관리 | 범용 재고 | 월 30,000~80,000원 | 치과 특화 아님 |

### 4.2 가격 포지셔닝
- **Standard 49,000원**: 범용 재고관리 SaaS 수준, 치과 전문 기능 포함으로 차별화
- **Premium 99,000원**: 덴트웹 대비 저렴하면서 임플란트 재고 특화 가치 제공
- **연 결제 20% 할인**: 고객 이탈률 감소 + 현금흐름 안정화

### 4.3 목표 매출 시나리오 (12개월 후)

| 항목 | 수치 | 산출 근거 |
|------|------|-----------|
| 가입 병원 수 | 200개 | 월 15~20개 신규 유입 |
| Free 유지 | 100개 (50%) | 체험 후 미전환 |
| Standard 전환 | 70개 (35%) | 핵심 수익층 |
| Premium 전환 | 25개 (12.5%) | 중형 치과 |
| Enterprise | 5개 (2.5%) | 네트워크 치과 |
| **월 매출 (MRR)** | **약 6,900,000원** | Standard 70 x 49K + Premium 25 x 99K + Enterprise 5 x 200K |
| **연 매출 (ARR)** | **약 82,800,000원** | MRR x 12 |

## 5. 전환 퍼널 설계

### 5.1 무료 → 유료 전환 전략

```
가입 → Free 체험(14일 Standard 기능 전체 개방)
  → 체험 종료 → Free 제한 적용
  → 기능 제한 시 업그레이드 유도 팝업
  → Standard 전환
```

### 5.2 전환 트리거 포인트

| 트리거 | 시점 | 유도 메시지 |
|--------|------|------------|
| 재고 50개 초과 | 품목 추가 시 | "Standard로 업그레이드하면 500개까지 관리할 수 있습니다" |
| 주문 관리 접근 | 탭 클릭 시 | "발주/주문 관리는 Standard 이상에서 사용 가능합니다" |
| FAIL 관리 접근 | 탭 클릭 시 | "FAIL 교환 추적은 Standard 이상에서 사용 가능합니다" |
| 3개월 데이터 초과 | 수술기록 업로드 시 | "전체 이력을 보관하려면 Standard로 업그레이드하세요" |
| 스태프 초대 | 구성원 관리 접근 시 | "팀 기능은 Premium에서 사용 가능합니다" |

### 5.3 체험 기간 정책
- **최초 가입 시**: 14일간 Standard 기능 전체 무료 체험
- **체험 종료 후**: 자동으로 Free 플랜으로 전환 (데이터 유지, 기능만 제한)
- **연 결제 유도**: "연 결제 시 2개월 무료" 프로모션

## 6. 결제 시스템 구현 방향

### 6.1 결제 수단
- **1순위**: 카드 정기결제 (토스페이먼츠 또는 아임포트)
- **2순위**: 계좌이체 (Enterprise 고객)
- **세금계산서**: 사업자 고객 대상 자동 발행

### 6.2 구현 우선순위

| 순서 | 항목 | 설명 |
|:----:|------|------|
| 1 | 플랜 상태 관리 | User/Hospital에 plan, planExpiry 필드 추가 |
| 2 | 기능 게이팅 | 플랜별 기능 접근 제어 로직 |
| 3 | 플랜 선택 UI | 가격표 페이지 + 플랜 변경 화면 |
| 4 | 결제 연동 | PG사 API 연동 (토스페이먼츠) |
| 5 | 구독 관리 | 자동 갱신, 해지, 다운그레이드 처리 |

### 6.3 필수 선행 조건
- **서버/DB 구축**: localStorage → 서버 API + DB 전환 (현재 미구현)
- **인증 보안 강화**: 평문 비밀번호 → bcrypt 해싱 + JWT 토큰
- **결제 사업자 등록**: PG사 계약, 사업자등록증 필요
