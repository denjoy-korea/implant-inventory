# 이벤트 계측 품질 점검표 (Quality Check Template)

> **목적**: 퍼널 이벤트 계측의 누락률·중복률·세션 분해 실패율을 주간 단위로 점검
> **점검 주기**: 매주 월요일 (또는 배포 직후)
> **소스**: `docs/04-report/traffic-kpi-daily/` 스냅샷 데이터 기반

---

## 점검 방법

1. 최근 7일 스냅샷 파일 확인: `traffic-kpi-snapshot-YYYY-MM-DD.md`
2. 아래 체크리스트 항목별로 수치 기입
3. 기준 미달 항목 → 원인 파악 후 `action` 컬럼에 기재
4. 완료 후 이 파일을 복사해 `quality-check-YYYY-MM-DD.md` 로 저장

---

## 주간 체크리스트

### 1. 세션 커버리지

| 항목 | 기준 | 이번 주 수치 | 통과 | 비고 |
|------|------|:-----------:|:----:|------|
| 유효 세션(session_id 보유) 수 | ≥ 300 | | | 누적 목표 |
| session_id 누락률 | < 1% | | | `(전체 이벤트 - session_id 있는 이벤트) / 전체 이벤트` |
| 일별 스냅샷 생성 여부 | 7/7일 | | | GitHub Actions 실패 알림으로 커버 |

### 3. CVR 이상 탐지

| 지표 | 정상 범위 | 이번 주 수치 | 통과 | 비고 |
|------|----------|:-----------:|:----:|------|
| 단계별 CVR (전 단계 대비) | 0% ~ 100% | | | 100% 초과 시 중복 이벤트 의심 |
| 결제 성공률 (start → success) | > 0% (실결제 전환 후) | | | 테스트 모드 중에는 N/A |
| 모바일 이탈률 | 모니터링 | | | 급등 시 UX 이슈 점검 |

### 4. 중복 이벤트 점검

| 항목 | 기준 | 이번 주 수치 | 통과 | 비고 |
|------|------|:-----------:|:----:|------|
| 동일 session_id + 동일 event_type 중복 건 | < 5% | | | 같은 세션에서 중복 발화 여부 |
| `auth_complete` 중복 (1세션 1회 기대) | < 1% | | | |

### 5. 데이터 적재 지연

| 항목 | 기준 | 통과 | 비고 |
|------|------|:----:|------|
| 스냅샷 생성 시각 vs KST 00:05 | ± 10분 이내 | | GitHub Actions 로그 확인 |
| Supabase → 스냅샷 반영 지연 | < 1시간 | | |
